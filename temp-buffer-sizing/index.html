    Noodle Temporary Buffer Sizing

    A distinctive feature of Noodle's convolution implementation is internal pooling. Instead of materializing the full unpooled convolution tensor, Noodle accumulates one output channel at a time, applies bias and activation, immediately performs pooling, and stores only the pooled output plane.

    Let \(C_\text{out}\) denote the number of convolution output channels and let \(V_\text{conv}\) denote the spatial width of one pre-pooling convolution output plane. A conventional implementation may require a temporary unpooled tensor of \(C_\text{out}V_\text{conv}^2\) elements, whereas Noodle requires only \(V_\text{conv}^2\) temporary elements for the current output plane.

    This note explains how to choose the correct size for the temporary buffers passed to:

    noodle_setup_temp_buffers(BUFFER1, BUFFER2);

    The main example is a LeNet-5-style MNIST model:

    Input: 28 x 28 x 1
Conv1: 5 x 5, P = 2, S = 1, output channels = 6
Pool1: 2 x 2, stride = 2
Conv2: 5 x 5, P = 0, S = 1, output channels = 16
Pool2: 2 x 2, stride = 2
Flatten: 5 x 5 x 16 = 400
FC1: 120
FC2: 84
FC3: 10

    1. The important distinction

    Noodle temporary buffers are not the same as layer output buffers.

    There are two different memory needs:

    Temporary buffers:
  scratch memory used internally while computing one layer

Layer output buffers:
  memory needed to store the result that will be consumed by the next layer

    For convolution with internal pooling, Noodle does not need to store the full unpooled tensor for all output channels. It only needs a temporary single output plane before pooling.

    So for one convolution layer:

    BUFFER2 size = Vconv x Vconv floats

    not:

    n_outputs x Vconv x Vconv floats

    This is the key memory-saving point.

    2. Compute convolution spatial size

    For normal Conv2D, the pre-pooling convolution width is:

    Vconv = (W - K + 2 * P) / S + 1;

    where:

    W = input width
K = kernel size
P = padding
S = stride

    After pooling:

    Vout = (Vconv - M) / T + 1;

    where:

    M = pooling window size
T = pooling stride

    For the usual 2 x 2 pooling with stride 2:

    Vout = Vconv / 2;

    assuming Vconv is even.

    3. LeNet-5 example

    Conv1

    Input W = 28
K = 5
P = 2
S = 1
Output channels = 6
Pool M = 2
Pool T = 2

    Pre-pooling convolution width:

    Vconv1 = (28 - 5 + 2*2) / 1 + 1
       = 28

    Pooled output width:

    Vout1 = (28 - 2) / 2 + 1
      = 14

    Temporary accumulator needed:

    BUFFER2 >= 28 x 28 floats
        >= 784 floats
        >= 3136 bytes

    Layer output storage needed:

    Conv1 output = 6 x 14 x 14 floats
             = 1176 floats
             = 4704 bytes

    This is not a temporary buffer. It is the actual output tensor after pooling.

    Conv2

    Input W = 14
K = 5
P = 0
S = 1
Output channels = 16
Pool M = 2
Pool T = 2

    Pre-pooling convolution width:

    Vconv2 = (14 - 5 + 2*0) / 1 + 1
       = 10

    Pooled output width:

    Vout2 = (10 - 2) / 2 + 1
      = 5

    Temporary accumulator needed:

    BUFFER2 >= 10 x 10 floats
        >= 100 floats
        >= 400 bytes

    Layer output storage needed:

    Conv2 output = 16 x 5 x 5 floats
             = 400 floats
             = 1600 bytes

    4. Choose the global temporary size

    Because the same temporary buffers are reused across layers, choose the maximum requirement across all convolution layers.

    For this LeNet-5 example:

    Conv1 BUFFER2 requirement = 28 x 28 = 784 floats
Conv2 BUFFER2 requirement = 10 x 10 = 100 floats

    Therefore:

    BUFFER2 must hold at least 784 floats.

    In bytes:

    784 x sizeof(float) = 784 x 4 = 3136 bytes

    So this is correct:

    static const int IMG_SIZE = 28 * 28;

float *BUFFER1 = (float *)malloc(IMG_SIZE * sizeof(float));
float *BUFFER2 = (float *)malloc(IMG_SIZE * sizeof(float));

noodle_setup_temp_buffers(BUFFER1, BUFFER2);

    Here both buffers are 784 floats, which is enough for this model.

    5. What BUFFER1 is for

    BUFFER1 is used as an input scratch buffer when Noodle streams an input tensor from a file.

    For file-input convolution, the rule is:

    BUFFER1 >= W x W elements

    For float file input:

    BUFFER1 >= W x W floats

    For byte file input:

    BUFFER1 >= W x W bytes

    For the LeNet-5 example, the maximum input width read by a convolution is:

    max W = 28

    Therefore:

    BUFFER1 >= 28 x 28 floats
        >= 784 floats
        >= 3136 bytes

    If the input tensor is already in memory, some convolution overloads do not require BUFFER1. However, allocating and passing both buffers is usually simpler and safer.

    6. Layer output buffers are separate

    When using memory output, the destination buffer must be large enough for the pooled output tensor:

    output buffer >= n_outputs x Vout x Vout floats

    For LeNet-5:

    Conv1 output = 6 x 14 x 14 = 1176 floats
Conv2 output = 16 x 5 x 5 = 400 floats
Flatten output = 400 floats
FC1 output = 120 floats
FC2 output = 84 floats
FC3 output = 10 floats

    So for a memory-to-memory LeNet-5 implementation, the largest true activation tensor is Conv1 output:

    6 x 14 x 14 = 1176 floats = 4704 bytes

    That is why a CYD-style memory example can use:

    BUFFER2 = malloc(28 * 28 * sizeof(float));       // temp accumulator
BUFFER3 = malloc(14 * 14 * 6 * sizeof(float));   // Conv1 pooled output

    The first one is temporary scratch. The second one is real layer output storage.

    7. File-backed version

    For file-backed convolution outputs, the large layer output tensors can be written to SD/flash instead of SRAM.

    For example:

    V = noodle_conv_float(GRID, 1, 6,
                      "out1.bin", 28,
                      cnn1, pool, NULL);

V = noodle_conv_float("out1.bin", 6, 16,
                      "out2.bin", V,
                      cnn2, pool, NULL);

    In this style, SRAM only needs:

    Input image / working buffer: 28 x 28 floats
BUFFER1:                  up to 28 x 28 floats
BUFFER2:                  up to 28 x 28 floats
FC buffers:               enough for flatten / dense outputs

    The Conv1 output tensor, 6 x 14 x 14, is stored in out1.bin instead of SRAM.

    8. Practical sizing checklist

    For each Conv2D layer, write down:

    W, K, P, S, n_outputs, pool M, pool T

    Then compute:

    Vconv = (W - K + 2P) / S + 1
Vout  = (Vconv - M) / T + 1

    Temporary buffer requirements:

    BUFFER1 >= max(W x W) for file-input convolution
BUFFER2 >= max(Vconv x Vconv) for convolution accumulator

    Output buffer requirement, only when output is stored in SRAM:

    output >= n_outputs x Vout x Vout floats

    For dense layers:

    input buffer  >= n_inputs floats
output buffer >= n_outputs floats

    The safest reusable SRAM buffers should be sized for the largest value that appears anywhere in the model.

    9. Final answer for the LeNet-5 example

    For this model:

    Conv1 pre-pool size: 28 x 28
Conv2 pre-pool size: 10 x 10
Largest pre-pool plane: 28 x 28 = 784 floats

    Therefore:

    BUFFER1 >= 784 floats   // file input scratch, when needed
BUFFER2 >= 784 floats   // convolution accumulator / pre-pooling plane

    In bytes:

    784 floats x 4 bytes = 3136 bytes per temporary buffer

    So this is correct:

    static const int TEMP_SIZE = 28 * 28;

float *BUFFER1 = (float *)malloc(TEMP_SIZE * sizeof(float));
float *BUFFER2 = (float *)malloc(TEMP_SIZE * sizeof(float));

noodle_setup_temp_buffers(BUFFER1, BUFFER2);

    For a memory-output version, also allocate enough storage for the largest real activation tensor:

    largest activation = Conv1 pooled output
                   = 6 x 14 x 14
                   = 1176 floats
                   = 4704 bytes

    For a file-output version, this activation can live on SD/flash instead.
